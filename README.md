A simple CLI tool to check if you have any pending GitHub PR review requests.

## Installation

```bash
go build -o sir
# Optionally move to your PATH
sudo mv sir /usr/local/bin/
```

## Usage

### 1. Configure Authentication

First, set up your GitHub authentication:

```bash
sir auth
```

You'll be prompted for:

- **GitHub host**: Press Enter for github.com, or enter your GitHub Enterprise host (e.g., github.ibm.com)
- **GitHub username**: Your GitHub username (e.g., owczr)
- **GitHub personal access token**: Your GitHub PAT with `repo` scope

### 2. Configure Repositories

Add repositories you want to monitor:

```bash
sir config
```

This opens an interactive menu where you can:

- Add repositories (format: `owner/repo`)
- Remove repositories
- View current configuration

### 3. Check for Review Requests

Simply run:

```bash
sir
```

This will check all configured repositories and display:

- "Nothing to review!" if there are no pending reviews
- A list of repositories with the number of PRs waiting for your review

### 4. Open PRs in Browser

To open a specific PR in your default browser:

```bash
sir open
```

This will:

1. List all PRs waiting for your review
2. Let you select which PR to open
3. Open the selected PR in your default browser

## Configuration

Configuration is stored in `~/.config/sir/config.json`

Example configuration:

```json
{
  "github_host": "github.com",
  "github_username": "owczr",
  "github_token": "ghp_xxxxxxxxxxxx",
  "repositories": [
    "owner/repo1",
    "owner/repo2"
  ]
}
```

## Requirements

- Go 1.16 or later
- GitHub personal access token with `repo` scope
- Internet connection to access GitHub API

## GitHub Enterprise Support

The tool supports GitHub Enterprise. During `sir auth`, simply enter your GitHub Enterprise host (e.g., `github.ibm.com`) instead of pressing Enter for the default github.com.
